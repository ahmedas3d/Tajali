## Summary

Wire the تجلي app's bootstrap sequence and 5-tab bottom navigation shell. `main()` initialises the Flutter binding, locks orientation to portrait, styles the status bar, then hands off to a Riverpod `ProviderScope` wrapping the root `TajaliApp` widget. A `selectedTabProvider` (Riverpod `StateProvider<int>`) drives `MainNavigation`, which uses an `IndexedStack` to preserve screen state across tab switches. Five placeholder screens — each a `StatelessWidget` with an AppBar titled `تَجَلِّي` and a centred Arabic section label — serve as mountpoints for later phases.

**Current state**: All 8 source files are already on disk from prior work. The only outstanding delta is that 4 of the 5 placeholder screens display their section name in the AppBar instead of `تَجَلِّي` (resolved in tasks).

## Design Decisions

### Navigation state ownership
`selectedTabProvider` is a top-level `StateProvider<int>` in `routes.dart`. It is intentionally global (not scoped to a widget) so that future features can read or mutate the active tab without needing a `BuildContext`. A `StateNotifierProvider` would add unnecessary overhead for a single integer.

### IndexedStack over PageView
`IndexedStack` keeps all 5 screen widgets alive and rendered off-screen, ensuring scroll positions and any future `StatefulWidget` state survive tab switches. `PageView` would require `AutomaticKeepAliveClientMixin` boilerplate on every screen.

### Theme mode
`app.dart` watches `themeProvider` (Phase 2) rather than hard-coding `ThemeMode.light`. This is an intentional forward-compatible choice: the Phase 2 theme provider already handles the `shared_preferences` persistence and defaults to `ThemeMode.light`, satisfying FR-010 while enabling the future user toggle without modifying `app.dart`.

### AppBar title
All 5 placeholder screens use `تَجَلِّي` as the AppBar title. The centred body text identifies the specific section. This matches the clarification from the 2026-06-25 session.

## Implementation Notes

`main()` calls `storageService.initialize()` before `runApp`. This is a Phase 2 dependency already wired in; Phase 3 makes no changes to it. The `StorageService` import and global variable declaration in `main.dart` are Phase 2 artefacts that remain.

The `BottomNavigationBar` uses `BottomNavigationBarType.fixed` (inherited from the theme's `bottomNavigationBarTheme`) which prevents label truncation on small screens and satisfies the edge case requirement.

`Directionality(textDirection: TextDirection.rtl)` is set at the `MainNavigation` level, which wraps the entire scaffold including the bottom bar. This ensures all descendants render RTL without each screen having to declare it.
