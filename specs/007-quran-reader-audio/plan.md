# Implementation Plan: Quran Reading & Audio (شاشة القراءة والتلاوة)

**Branch**: `007-quran-reader-audio` | **Date**: 2026-06-27 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/007-quran-reader-audio/spec.md`

---

## Summary

Replace the `SurahStubScreen` placeholder with a full `QuranReaderScreen` that displays Uthmanic Arabic text using the bundled `AmiriQuran` font and plays per-ayah audio via `just_audio`'s `ConcatenatingAudioSource`. Audio is scoped to the screen lifecycle (no background service). Ayah-level bookmarks, font size preference, reciter selection, and per-surah reading position persistence are layered on top.

---

## Technical Context

**Language/Version**: Dart 3.3 / Flutter 3.19+

**Primary Dependencies**:
- `just_audio ^0.9.38` (already in pubspec) — audio player with `ConcatenatingAudioSource` + `currentIndexStream`
- `hive_flutter ^1.1.0` (already in pubspec) — ayah text cache + ayah bookmarks
- `shared_preferences ^2.2.3` (already in pubspec) — reciter, font size, reading position
- `dio ^5.4.3` (already in pubspec) — HTTP fetch from AlQuran Cloud
- `flutter_riverpod ^2.5.1` (already in pubspec) — state management

**Storage**:
- Hive box `ayahTextBox` — `List<AyahModel>` per surah, keyed `surah_text_{n}`
- Hive box `ayahBookmarksBox` — `List<AyahBookmark>`, ayah-level bookmarks
- SharedPreferences — reciter (`quran_selected_reciter`), font size (`quran_font_size`), per-surah position (`reader_pos_{n}`)

**Testing**: `flutter_test` (widget + unit); golden tests for reader screen

**Target Platform**: iOS + Android (existing targets)

**Performance Goals**:
- Cached surah open: < 300ms (SC-004)
- First load (network): < 2s visible text (SC-001)
- Audio start: < 1.5s (SC-002)
- Highlight advance: < 200ms lag (SC-003)

**Constraints**:
- No background audio service (FR-019)
- Audio streamed on demand; no local audio caching
- AmiriQuran font only (already bundled — no new font assets)
- New Hive typeIds: AyahModel = 13, AyahBookmark = 14

**Scale/Scope**: 114 surahs, 6,236 ayahs total; only one surah loaded at a time

## Implementation Phases

### Phase A — Data Layer

**A1. `AyahModel`** (`lib/features/quran/data/models/ayah_model.dart`)
- Hive object, typeId 13
- Fields: `number`, `numberInSurah`, `surahNumber`, `text`, `juz`, `page`, `audioUrl`
- `fromJson()` factory — parses AlQuran Cloud `quran-uthmani` edition ayah object
- Run `build_runner` to generate `ayah_model.g.dart`

**A2. `AyahBookmark`** (`lib/features/quran/data/models/ayah_bookmark.dart`)
- Hive object, typeId 14
- Fields: `surahNumber`, `ayahNumberInSurah`, `surahName`, `ayahText`, `createdAt`
- Override `==` and `hashCode` on `(surahNumber, ayahNumberInSurah)`
- Run `build_runner` to generate `ayah_bookmark.g.dart`

**A3. `ReciterModel`** (`lib/features/quran/data/models/reciter_model.dart`)
- Pure Dart, no Hive
- Static `const List<ReciterModel> reciters` with 4 entries (see [data-model.md](data-model.md))

**A4. `QuranReaderService`** (`lib/features/quran/data/services/quran_reader_service.dart`)
- `Future<List<AyahModel>> getSurahWithAudio(int surahNumber, String reciterEdition)`:
  - Check Hive cache first (`ayahTextBox`, key `surah_text_{surahNumber}`)
  - If miss: call `GET /v1/surah/{n}/editions/quran-uthmani,{reciter}`
  - Parse both editions, zip by `numberInSurah`, build `List<AyahModel>`
  - Store in cache before returning
- `Future<List<AyahModel>> updateAudioUrls(List<AyahModel> ayahs, String reciterEdition)`:
  - Called when reciter changes after text is already cached
  - Calls `GET /v1/surah/{n}/{reciterEdition}`, updates `audioUrl` on each `AyahModel`
  - Re-saves to Hive cache

**A5. `AyahBookmarkService`** (`lib/features/quran/data/services/ayah_bookmark_service.dart`)
- `List<AyahBookmark> loadAll()` — read from `ayahBookmarksBox`
- `void toggle(AyahBookmark bookmark)` — add or remove (by composite key)
- `bool isBookmarked(int surahNumber, int ayahNumberInSurah)` — quick lookup

**A6. `main.dart` modifications**:
- `Hive.openBox<AyahModel>('ayahTextBox')`
- `Hive.openBox<AyahBookmark>('ayahBookmarksBox')`
- `Hive.registerAdapter(AyahModelAdapter())`
- `Hive.registerAdapter(AyahBookmarkAdapter())`

### Phase B — State Layer

**B1. `reader_providers.dart`** (`lib/features/quran/providers/reader_providers.dart`)

New providers:

| Provider | Type | Default | Notes |
|---|---|---|---|
| `currentSurahProvider` | `StateProvider<int>` | 1 | Active surah number |
| `selectedReciterProvider` | `StateProvider<String>` | `'ar.alafasy'` | Loads from SharedPreferences on init |
| `fontSizeProvider` | `StateProvider<double>` | `28.0` | Loads from SharedPreferences on init |
| `surahAyahsProvider` | `FutureProvider.family<List<AyahModel>, (int, String)>` | — | Keyed by (surahNumber, reciterEdition) |
| `currentPlayingAyahProvider` | `StateProvider<int?>` | null | 0-based index in surah |
| `repeatModeProvider` | `StateProvider<bool>` | false | Single-ayah repeat toggle |
| `ayahBookmarksProvider` | `StateNotifierProvider<AyahBookmarksNotifier, List<AyahBookmark>>` | [] | Wraps `AyahBookmarkService` |

**B2. `AudioPlayerService`**:
- Wraps a `just_audio.AudioPlayer` instance
- `play(List<AyahModel> ayahs, {int startIndex = 0})` — builds `ConcatenatingAudioSource`, starts from `startIndex`
- `playFromIndex(int index)` — seek to playlist position
- `pause()` / `stop()` / `dispose()`
- `setRepeat(bool on, int ayahIndex)` — replaces source with looping single-item or restores playlist
- Exposes `currentIndexStream` → used by reader screen to drive `currentPlayingAyahProvider`

**B3. `quran_providers.dart` modifications**:
- Add `writeLastRead(int surahNumber, int ayahNumber)` helper (writes `quran_last_read_surah` + `quran_last_read_ayah`)
- Add `readingPositionProvider` — `FutureProvider.family<int, int>` — reads `reader_pos_{surahNumber}`
- Add `writeReadingPosition(int surahNumber, int ayahIndex)` helper

### Phase C — UI Layer

**C1. `QuranReaderScreen`** (`lib/features/quran/presentation/quran_reader_screen.dart`)

Constructor:
```
QuranReaderScreen({ required SurahModel surah, int? initialAyahIndex })
```

Layout:
- `AppBar`: surah name (AmiriQuran, gold), prev/next surah arrows, font size icon button
- `Body`: RTL `ListView.builder` of `AyahCard` widgets; Basmala header widget at top (except surah 9)
- `BottomBar`: `AudioPlayerBar`

Behaviour:
- On mount: scroll to `initialAyahIndex ?? savedReadingPosition`; subscribe to `currentIndexStream`
- On scroll: write per-surah reading position when visible ayah changes
- On current ayah crossing mid-screen: write global last-read keys
- On pop: stop audio; dispose player

**C2. `AyahCard`** (`lib/features/quran/presentation/widgets/ayah_card.dart`)

Props: `AyahModel ayah`, `bool isPlaying`, `bool isBookmarked`, `VoidCallback onTap`, `VoidCallback onBookmarkToggle`

Visual states:
- Default: green background, gold ayah marker, ivory AmiriQuran text
- Playing: gold border + `AppColors.gold.withOpacity(0.12)` background tint
- Bookmarked: filled bookmark icon (gold) in top-right corner of card

**C3. `AudioPlayerBar`** (`lib/features/quran/presentation/widgets/audio_player_bar.dart`)

RTL layout (right to left in reading order):
- Reciter chip (tappable → `ReciterPickerSheet`)
- Prev ayah button
- Play/Pause (large gold icon)
- Next ayah button
- Repeat toggle (`repeat_one` icon when active / `repeat` when off)
- Buffering overlay on play button when `PlayerState.loading`

**C4. `ReciterPickerSheet`** (`lib/features/quran/presentation/widgets/reciter_picker_sheet.dart`)
- Modal bottom sheet, lists 4 reciters with Arabic name + English name
- Current reciter highlighted in gold
- On selection: update `selectedReciterProvider`, persist to SharedPreferences, trigger `updateAudioUrls`

**C5. Font size control**:
- Three-button row (ص / ص / ص at sizes S/M/L) accessible from app bar icon button
- Shows as a `PopupMenuButton` or small `BottomSheet`
- Updates `fontSizeProvider` and persists to SharedPreferences

## Key Design Decisions (from research.md)

| Decision | Choice | Rationale |
|---|---|---|
| Audio package | `just_audio` ConcatenatingAudioSource | `currentIndexStream` gives free ayah sync |
| Quran font | `AmiriQuran` (existing asset) | Already bundled; Quran-specific glyphs |
| API strategy | Combined editions endpoint | One round-trip for text + audio |
| Text cache | Hive `ayahTextBox`, permanent | Quran text never changes |
| Ayah bookmarks | New Hive box, separate from surah bookmarks | Avoids schema migration risk |
| Reading position | SharedPreferences per-surah | Small integers; no Hive needed |
| Audio lifecycle | Screen-scoped, stops on pop | FR-019; no background service complexity |
| Repeat mode | Single-ayah via source swap | FR-020; full-surah loop deferred |
| Bookmark deep-link | `initialAyahIndex` param | FR-015a; clean, no global state |
